Tabeller

Det finns många fall att vilja göra en tabell, t.ex. för Sportresultat, deltagarlistor, anmälda till en utbildning, varor till salu, kalkyler, börskurser osv. En tabell är på sätt och vis en lista som bara vuxit till sig och fått två dimensioner.

Grunden

Vi skapar oss en tabell över kända karaktärer ifrån serietidningsvärlden.
Tabellen består av en rubrikrad och fyra vanliga rader men information placerad i två kolumner. Detta är alltså en väldigt enkel tabell som demonstreras. Nedan ser HTML-koden.

<table>
  <thead>
    <tr>
      <th>Förnamn</th>
      <th>Efternamn</th>
      <th>Ålder</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Kalle</td>
      <td>Anka</td>
      <td>24</td>
    </tr>
    <tr>
      <td>Kajsa</td>
      <td>Anka</td>
      <td>24</td>
    </tr>
    <tr>
      <td>Musse</td>
      <td>Pigg</td>
      <td>28</td>
    </tr>
    <tr>
      <td>Mimmi</td>
      <td>Pigg</td>
      <td>28</td>
    </tr>
  </tbody>
</table>

En tabell omsluts alltid av start- och sluttaggen <table></table> och innebörden är svår att missta sig på.

<thead>, <tbody>, <tfoot>

Alla tabeller har även ett huvud, <thead>, en kropp, <tbody>, och en fot, <tfoot>, även om taggarna skrivs med eller ej. Utelämnas taggarna så skapas de per automatik av webbläsaren.

<thead> används för att gruppera rubrikrader och tabellen kan ha en eller flera.

Även <tbody> kan finnas som en eller flera i en tabell och används för att gruppera. Man kan sortera en serietabell efter slutspelsplats, vanliga och nedflyttningsplats.

<tfoot> används med fördel för rader där det förekommer slutsummor eller andra avslutande uppgifter.

<tr>

Varje rad i tabellen motsvaras av elementet <tr> - table row.

<th>

Det finns två slags tabellceller (table cells), vanliga data-celler (nästa del) och rubrikceller. Normalt villa man åtminstone använda kolumnrubriker. En enkel <th> - table heading - som i koden ovan blir kolumnrubrik för den kolumn där den står.
Texten i <th> blir normalt fet och centrerad.
Vid utskrift ska en bra webbläsare kunna upprepa kolumnrubrikerna på varje sida, om tabellen är så pass stor att den inte får plats på ett papper.
Man kan också ange rubriker för enskilda rader. Det görs med attributet scope.
<th scope="row">
Scope kan sättas till row, col, rowgroup eller colgroup, men col är alltså default (i praktiken) och därför onödigt att skriva ut. (Formellt är default-värdet auto.)

<td>

Dataceller - table data - skrivs med taggen <td>.
Ibland vill man att en cell skall sträcka sig över flera rader eller kolumner. Det kan åstadkommas med attributen colspan och rowspan.

Andra element och attribut för tabeller

För vårt enkla exempel behövs inga fler element, men ibland kan man behöva ge ytterligare struktur eller information.
Med elementet <caption> sätter man en tabellrubrik för hela tabellen.
Med elementen <col> och <colgroup> avser en kolumn eller grupp kolumner. Bara ett fåtal CSS-egenskaper kan styras utifrån dessa element, så dessa används ganska sällan.
Med kombinationen av attributen axis, header och id kan man styra hur en tabells celler associeras med table header, när strukturen blir så komplex att attributen scope inte räcker till för jobbet. Å andra sidan, skulle dessa behövas är det kanske bättre att förenkla tabellens struktur.

Tabeller och CSS

Grundinställningarna för en tabell och dess celler är dess:

  • En tabell och dess celler är normalt så stora som det krävs för att visa innehållet.
  • Bredden på kolumner justeras automatiskt efter innehållet. Radbrytning sker om bredden kommer upp till 100%.
  • Höjden på raderna justeras automatiskt efter innehållet.
  • Varken tabell eller celler har kantlinjer.
  • Det finns ett avstånd - border-spacing - mellan cellerna.
Här kommer det exempel vi skrev tidigare, förutom att vi tilldelat tabellen två klasser,
<table class="borderless services"> så ser HTML-koden likadan ut.

Förnamn Efternamn Ålder
Kalle Anka 24
Kajsa Anka 24
Musse Pigg 28
Mimmi Pigg 28

För att få vår tabell att bli så fin så behövs det sättas en stil. Jag vill gärna att tabellen hålls ganska stilren och det gör jag först och främst med schemat nedan.

table {
  margin-bottom: 1.4em;
  width: 100%;
}
.borderless {
  *border-collapse: collapse; /* Fulhack för IE7 och lägre */
  border-spacing: 0;
}
.services {
  border: 1px solid #333;
  background: #f4f6f7;
  width: 96%;
  margin: 5% auto 2em;
  border-radius: 5px;
  box-shadow: 0 0 4px rgba(0, 0, 0, 0.2);
  text-shadow: #fff 1px 1px 1px;
}
thead {
  font-size: 150%;
}
th, td, caption {
  padding: 18px 28px 18px;
}
th {
  padding-top: 22px;
  background: #ddd;
}
td {
  border-top: 1px solid #333;
  border-right: 1px solid #333;
}

Markera utpekade rader

Även om en tabell är väldigt ren så kan vissa effekter vara behövliga. Ett gott exempel är att "highlighta" en rad om man pekar på den med muspekaren eller om den är ett mål för intern länkning. För att :hover skall vinna över :target så sätter vi den sist av de två.
Vi mjukar även upp färgskiftningen så att det blir behagligare att se med intoning via egenskapen transition.

tr {
  -webkit-transition: background-color 2s;
  -moz-transition: background-color 2s;
  -ms-transition: background-color 2s;
  -o-transition: background-color 2s;
}
tr:target {
  background-color: #056571;
}
tr:hover {
  background-color: #FF6A5C;
}

Zebra

Ett annat sätt att hjälpa ögat följa rader är att ge varannan rad en annan bakgrundsfärg. Detta kallas zebra-striping. Följande kod är inte använd på tabellen i exemplet, men illustrerar hur det görs.

tr:nth-child(odd) {
  background-color: #61D7A4;
}
tr:nth-child(even) {
  background-color: #FF9B73;
}
